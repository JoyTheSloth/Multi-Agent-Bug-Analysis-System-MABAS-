=============================================================
     SILK ANALYSIS - INSTRUCTIONS & SAMPLE PAYLOADS
=============================================================

This application features a full React frontend that connects 
to a Python FastAPI backend which orchestrates AI agents using Groq.

-------------------------------------------------------------
1. HOW TO RUN THE APPLICATION
-------------------------------------------------------------

STEP A: Start the Backend (API & LLM Agents)
1. Open a new terminal.
2. Navigate to the backend directory:
   cd backend
3. Create and activate a Virtual Environment (if not done yet):
   python -m venv venv
   # On Windows:
   venv\Scripts\activate
   # On Mac/Linux:
   source venv/bin/activate
4. Install dependencies into the virtual environment:
   uv pip install fastapi uvicorn groq python-dotenv pydantic
5. Run the FastAPI server:
   uv run uvicorn backend.main:app --port 8000

STEP B: Start the Frontend (Vite + React)
1. Open a second terminal window.
2. Ensure you are in the root directory (/silk-analysis).
3. If not installed already, install Node dependencies:
   npm install
4. Start the frontend:
   npm run dev
5. Open your web browser and go to http://localhost:3000

screen. You can use the following sample inputs to force the system to 
generate a Reproduction Script that successfully fails:

It tries to calculate the sum of an empty tuple or string instead of a valid array of integers, which crashes the script because of an unsupported operand type for +: 'int' and 'str'.

-------------------------------------------------------------
3. WHAT TO EXPECT
-------------------------------------------------------------
When you click "Launch Full Agentic Analysis":
1. The UI will show live Traceability Logs (e.g. "[Triage] Processing").
2. The Backend will use Groq to write a Python script based on your payload.
3. The Backend will physically execute that Python script.
4. Because the bugs describe fatal crashes, the generated script will throw a traceback error.
5. The UI will display the generated code, capture the raw 'Red' console error in the window, and provide a patch plan!
